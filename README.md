# Full-Stack Microservices E-Commerce Platform

## 🚀 Project Overview
This project is a scalable, end-to-end online retail solution designed with a **Microservices architecture**.The primary goal was to transition from a monolithic design to a resilient system where core business functions—such as user management, product catalogs, and order processing—operate as independent services.The platform manages the entire shopping lifecycle, from product discovery to secure checkout, demonstrating robust backend orchestration and a dynamic frontend.

## 🛠️ Tech Stack
* **Backend:** Java, Spring Boot, Microservices.
* **Database:** MySQL (Relational schema design for complex data mapping).
* **Communication:** REST APIs for inter-service and client-server data exchange.
* **Frontend:** JavaScript, HTML5, CSS3 (Responsive UI development).
* **Tools:** Git, GitHub, IntelliJ IDEA, VS Code.

## ✨ Core Features
* **Microservices Architecture:** Decoupled services for **User Management**, **Product Catalog**, and **Order Processing** to ensure high availability and independent scaling.
* **Secure Cart & Order Management:** Logic-heavy modules built to handle real-time user selections and transaction processing.
* **Payment Gateway Integration:** A simulated end-to-end payment flow providing a realistic checkout experience.
* **Dynamic UI:** A responsive frontend utilizing RESTful consumption for live, asynchronous data rendering.

## 🧠 Key Learnings & Implementation
As a developer, this project allowed me to implement several advanced software engineering concepts:
* **Service Discovery & Configuration:** Managed service communication and centralized settings within a distributed system.
* **Transaction Management:** Ensured data consistency across multiple database instances during the order lifecycle.
* **OOP Principles:** Applied a strong foundation in Object-Oriented Programming for modular and maintainable code.
* **Error Handling:** Implemented dynamic error handling to improve application stability and user experience.
